# Remotion 4.0.489 working notes (researched 2026-07-12)

## Findings
- VERSION: remotion 4.0.489 is latest on npm (confirmed via `npm view remotion version`, 2026-07-12). All @remotion/* packages version-lock to the exact same version — never mix versions across remotion, @remotion/cli, @remotion/transitions, @remotion/google-fonts, @remotion/zod-types.
- CORE HOOKS: `import {useCurrentFrame, useVideoConfig} from 'remotion'`. useCurrentFrame() returns the frame number relative to the enclosing <Sequence> (starts at 0 inside each Sequence — this is the key composition primitive). useVideoConfig() returns {fps, durationInFrames, width, height}. First frame is 0, last is durationInFrames-1.
- spring(): `import {spring} from 'remotion'`. Signature: spring({frame, fps, from?=0, to?=1, config?, durationInFrames?, delay?, reverse?, durationRestThreshold?}). config: {mass=1, stiffness=100, damping=10, overshootClamping=false}. damping:200 = smooth no-bounce (the go-to for UI motion); default damping:10 is bouncy. durationInFrames stretches the curve to an exact length. Delay via `delay: 20` or `frame: frame - 20` (both work; negative frame input yields the `from` value).
- interpolate(): `import {interpolate, Easing} from 'remotion'`. interpolate(inputValue, inputRange, outputRange, {easing?, extrapolateLeft?, extrapolateRight?}). Extrapolate values: 'extend' (default), 'clamp', 'wrap', 'identity' — almost always pass {extrapolateLeft:'clamp', extrapolateRight:'clamp'} for opacity/scale. inputRange must be monotonically increasing. Easing: Easing.bezier(x1,y1,x2,y2), Easing.in/out/inOut(Easing.cubic|quad|exp|sin), or an array of easings per segment. Since 4.0.472 output ranges can be CSS transform strings ('0deg'→'90deg', '0px 0px'→'100px 50px'). Common combo: pass a spring() result as inputValue with [0,1] input range to remap spring to any output range.
- AbsoluteFill: `import {AbsoluteFill} from 'remotion'` — a div with position:absolute inset 0, display:flex. Standard scene root; nest them to layer (later siblings render on top).
- Sequence: `import {Sequence} from 'remotion'`. Props: from (start frame), durationInFrames, layout ('absolute-fill' default | 'none'), premountFor, name, style. Children see a rebased useCurrentFrame() starting at 0 when the Sequence begins. Series: `import {Series} from 'remotion'` — <Series><Series.Sequence durationInFrames={40}>...</Series.Sequence>...</Series> plays children back-to-back; offset prop shifts (negative = overlap previous); only the LAST Series.Sequence may be Infinity.
- TRANSITIONS: `import {TransitionSeries, linearTiming, springTiming} from '@remotion/transitions'`; presentations from subpaths: `import {fade} from '@remotion/transitions/fade'`, `import {slide} from '@remotion/transitions/slide'`, `import {wipe} from '@remotion/transitions/wipe'`. Pattern: <TransitionSeries><TransitionSeries.Sequence durationInFrames={90}><SceneA/></TransitionSeries.Sequence><TransitionSeries.Transition presentation={fade()} timing={linearTiming({durationInFrames: 15})}/><TransitionSeries.Sequence durationInFrames={120}><SceneB/></TransitionSeries.Sequence></TransitionSeries>. timing options: linearTiming({durationInFrames, easing?}) and springTiming({config?, durationInFrames?, durationRestThreshold?}). slide({direction: 'from-left'|'from-right'|'from-top'|'from-bottom', enterStyle?, exitStyle?}); presentation defaults to slide() if omitted. TransitionSeries.Overlay (4.0.415+) renders content centered at a cut without changing timeline math.
- TRANSITION DURATION MATH (critical for a 60-90s promo): transitions OVERLAP scenes, so composition durationInFrames = sum(scene durations) - sum(transition durations). E.g. scenes 40+60+90 with transitions 30+45 = 115 frames total. Get a timing's length programmatically: `timing.getDurationInFrames({fps})`. A transition's duration must not exceed either adjacent sequence's duration; no two consecutive Transitions allowed.
- CUSTOM PRESENTATION: a function returning `TransitionPresentation<Props>` = {component, props}. The component receives {children, presentationDirection: 'entering'|'exiting', presentationProgress: 0..1, passedProps}. Wrap children in an AbsoluteFill and apply clipPath/transform/opacity keyed on progress; typically no-op when presentationDirection==='exiting'. Type import: `import type {TransitionPresentation, TransitionPresentationComponentProps} from '@remotion/transitions'`.
- FONTS (Figtree): `import {loadFont} from '@remotion/google-fonts/Figtree'`. Call at MODULE TOP LEVEL (not inside a component): `const {fontFamily, waitUntilDone} = loadFont('normal', {weights: ['400','600','700'], subsets: ['latin']})` — weights are STRINGS. Then `style={{fontFamily}}`. loadFont blocks the render until the font is ready (internally uses delayRender), so no FOUT in output. Always narrow weights+subsets — bare loadFont() pulls every variant and can hit the 30s delayRender timeout. Typesafe: invalid weights fail at compile time.
- staticFile: `import {staticFile} from 'remotion'`. Put assets in `public/` at project root; reference as staticFile('logo.png') (no leading slash, no 'public/' prefix). Use for <Img src={staticFile(...)}/>, <OffthreadVideo/>, <Audio/>, fonts. Never use relative paths or import statements for public assets.
- COMPOSITION REGISTRATION: src/index.ts contains `import {registerRoot} from 'remotion'; registerRoot(RemotionRoot);`. src/Root.tsx exports RemotionRoot returning <Composition id="Promo" component={Promo} durationInFrames={2250} fps={30} width={1920} height={1080} schema={promoSchema} defaultProps={{...}}/>. id must be unique, [a-zA-Z0-9-] only (it's the CLI render target). Multiple <Composition>s go in a fragment; group with <Folder name="..."> in Studio. For a 75s promo at 30fps: durationInFrames = 2250.
- ZOD PROPS: `import {z} from 'zod'`. `export const promoSchema = z.object({title: z.string(), accent: zColor()})`; component typed as React.FC<z.infer<typeof promoSchema>>; pass schema + matching defaultProps to <Composition>. `import {zColor} from '@remotion/zod-types'` gives a color picker in Studio; also zTextarea(), zMatrix(). Install via `npx remotion add @remotion/zod-types zod` so the zod version matches what Remotion supports (Remotion pins a specific zod v3 line — do not hand-pin zod 4 without verifying). defaultProps must be complete (no partials). Props become editable live in Remotion Studio's right panel.
- calculateMetadata: pass `calculateMetadata={async ({props}) => ({durationInFrames, fps, props})}` on <Composition> to derive duration/fps/props from data (e.g. video length, API fetch). It runs once, NOT per-frame — the one place true randomness/fetching is safe.
- remotion.config.ts (project root): `import {Config} from '@remotion/cli/config';` then Config.setVideoImageFormat('jpeg'), Config.setOverwriteOutput(true), Config.setConcurrency(8), Config.setEntryPoint('./src/index.ts'), Config.setPublicDir(...), Config.overrideWebpackConfig(cfg => ...). Applies to CLI (studio/render/still) ONLY — has no effect on @remotion/renderer SSR APIs or <Player>.
- CLI: `npx remotion studio` (dev preview at localhost:3000), `npx remotion render <entry?> <composition-id> <output>` e.g. `npx remotion render Promo out/promo.mp4`. Key render flags: --codec=h264(default)|h265|vp9|prores|png, --props='{...}' or --props=./props.json (file form required on Windows), --frames=0-100, --concurrency, --crf, --scale (0-16), --jpeg-quality, --muted, --timeout (delayRender timeout, default 30000ms), --log=verbose. Still: `npx remotion still <entry?> <comp-id> <out.png>` with --frame=N, --image-format=png|jpeg, --scale. `npx remotion add <pkg>` installs version-matched Remotion packages.
- DETERMINISM: rendering spins up MANY parallel headless browser instances; every frame must be a pure function of frame number + props. Use `import {random} from 'remotion'; random('seed-'+i)` for pseudorandom (same seed → same value everywhere, returns 0..1). random(null) opts into true randomness (rarely correct). No Math.random(), no Date.now()/new Date() in render code, no useState-driven animation, no CSS transitions/animations, no requestAnimationFrame — animate exclusively from useCurrentFrame().
- PREMOUNTING: `<Sequence premountFor={60}>` (also on Series.Sequence and TransitionSeries.Sequence) mounts children early with opacity:0, pointer-events:none and frozen time so videos/images/fonts load before appearing — fixes flicker in <Player> and helps heavy assets at render. Not available with layout="none". In v4 it is opt-in (v5 will default to ~1s). prefetch(): `import {prefetch} from 'remotion'; const {free, waitUntilDone} = prefetch(staticFile('clip.mp4'))` — Player-only optimization (fetches into blob URL); irrelevant for CLI renders. For embedded video during renders prefer <OffthreadVideo> over <Video>.
- MULTI-SCENE 60-90s PROMO STRUCTURE (recommended layout): root: remotion.config.ts, public/ (images, audio, video), src/index.ts (registerRoot), src/Root.tsx (Composition registration + schema), src/Promo.tsx (top-level comp = one TransitionSeries over scenes), src/scenes/Intro.tsx, Feature1.tsx, ..., Outro.tsx (one file per scene, each rooted in AbsoluteFill, each animating from its LOCAL frame 0), src/components/ (reusable animated bits: AnimatedText, LogoReveal), src/lib/theme.ts (colors, fontFamily from loadFont, spacing constants), src/lib/timing.ts (scene duration constants in frames + helper computing total = sum(scenes) - sum(transitions), exported so Root.tsx's durationInFrames can never drift from the scene list). Register each scene as its own extra <Composition> too — lets you iterate on one scene in Studio and render stills of it for review. Audio: place one <Audio src={staticFile('music.mp3')} volume={...}/> at the Promo root, outside the TransitionSeries.
- SCENE AUTHORING PATTERN: each scene = React.FC rooted in <AbsoluteFill style={{backgroundColor, fontFamily}}>; entrance = spring({frame, fps, config:{damping:200}}) driving translateY/opacity; stagger children with delay or frame - i*4; exit animations usually unnecessary when TransitionSeries presentations handle the handoff. Interpolate spring output: `const y = interpolate(enter, [0,1], [40, 0])`.
- PERFORMANCE: JPEG frame format is fastest (default); use --concurrency near CPU core count; keep delayRender usage minimal and always paired with continueRender (loadFont/Img/OffthreadVideo handle this internally); wrap expensive pure computation in useMemo keyed on frame-independent inputs only.

## Do not
- Do NOT use Math.random(), Date.now(), new Date(), performance.now(), or crypto.randomUUID() in component render code — parallel render workers each get different values, producing corrupted/flickering output. Use random(seed) from 'remotion' and derive all time from useCurrentFrame().
- Do NOT animate with useState/useEffect timers, CSS transitions, CSS @keyframes animations, or requestAnimationFrame — Remotion screenshots isolated frames, so anything not derived from useCurrentFrame() renders frozen or inconsistently.
- Do NOT forget {extrapolateLeft:'clamp', extrapolateRight:'clamp'} on interpolate() for opacity/scale — default 'extend' extrapolates past the range (opacity > 1, negative scale) before/after the animation window.
- Do NOT call spring() without passing fps from useVideoConfig() — hardcoding fps desyncs the animation if the composition fps changes.
- Do NOT compute the Composition durationInFrames as the plain sum of scene durations when using TransitionSeries — transitions overlap scenes; total = sum(scenes) - sum(transitions). Getting this wrong leaves black frames at the end or truncates the last scene.
- Do NOT make a transition longer than either adjacent TransitionSeries.Sequence, place two TransitionSeries.Transition components consecutively, or mix a Transition and an Overlay between the same pair of sequences — all throw at runtime.
- Do NOT call loadFont() inside a component body — call it once at module top level. And do NOT call loadFont() with no arguments (loads every weight/style/subset → dozens of network requests → delayRender 30s timeout). Pass style + {weights: ['400','700'], subsets: ['latin']}; weights are strings, not numbers.
- Do NOT reference public/ assets with relative paths, absolute paths, leading slashes, or ES imports — use staticFile('name.ext'). staticFile('/logo.png') and staticFile('public/logo.png') are both wrong.
- Do NOT use <Video> for embedded video in CLI/SSR renders — use <OffthreadVideo> (frame-accurate extraction; <Video> can drop/repeat frames during rendering).
- Do NOT mix versions of remotion packages (e.g. remotion@4.0.489 with @remotion/transitions@4.0.480) — exact version match is required; use `npx remotion add <pkg>` or `npx remotion upgrade`.
- Do NOT hand-pin zod (especially zod v4) for schema props — install via `npx remotion add zod @remotion/zod-types` to get the version line Remotion's Studio prop editor supports (zod v3).
- Do NOT give durationInFrames={Infinity} to any Series.Sequence except the last one, and do NOT omit durationInFrames on TransitionSeries.Sequence — it is required there.
- Do NOT expect remotion.config.ts to affect @remotion/renderer SSR APIs or the <Player> — it configures the CLI only; SSR takes options as function parameters.
- Do NOT rely on defaultProps for render-time data variation — override at render with --props (and on Windows, --props must be a file path, inline JSON is not supported).
- Do NOT provide partial defaultProps when a schema is set — defaultProps must fully satisfy the zod schema or the composition fails validation in Studio.
- Do NOT use premountFor on a Sequence with layout="none" — unsupported. And do NOT use prefetch() expecting it to speed up CLI renders — it is a Player-only optimization.
- Do NOT put composition ids with spaces/special chars — ids must match [a-zA-Z0-9-] since they are CLI arguments.
- Do NOT fetch data or run true randomness per-frame — the only safe place for nondeterminism/async data is calculateMetadata (runs once) or delayRender/continueRender pairs that resolve identically on every worker.
